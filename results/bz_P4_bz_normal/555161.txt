In some cases, a property can be decomposed by many other properties:
For instance, a property providing a list of elements can lead to have "sub-properties" to each element of the list returned by a property.

This is the case in org.polarsys.capella.common.re.re2rpl.create.properties.ReplicaContentProperty, describing the content of a replica and sub properties should have been the (user-choice-made location, default location, implicit location) of the element in the model which are now stored as hashMap within a ILocationHandler
