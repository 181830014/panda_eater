Recently I upgraded my project from Java 8 to Java 11, including a switch from Gradle 4 to Gradle 5.

Everything works fine with one exception:

In some of my projects I cannot execute a test class anymore:

- Go to class containing JUnit 4 tests. Place the cursor somewhere in the class.
- Choose "Run as JUnit Test"
=> Error message "No JUnit tests found"

Executing single tests from this class works and executing the whole test class also works if the name of the test class is marked.

The same problem occurs at the project level: If I select a project and choose "Run as JUnit Test" then some projects just show the "No JUnit test found".

Not 100% sure if buildship is actually the component responsible for the problem, but at least I found multiple buildship-related error messages in the Eclipse log (see below).

I could not identify any meaningful difference between projects where this works and projects where this does not work. All projects have a similar .classpath and .project, both generated by buildship.

Any ideas?
