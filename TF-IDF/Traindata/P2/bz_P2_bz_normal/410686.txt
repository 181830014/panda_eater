ClassDescriptor#checkDatabase(AbstractSession) should use 'databaseFields.contains(field.getNameForComparisons())' instead of 'databaseFields.contains(field.getName())' on line #953 (in EclipseLink 2.5.0-M6.

Otherwise, database validation will always fail for mixed-case, differing column names with 'eclipselink.jpa.uppercase-column-names=true' because 'databaseFields' contains all-uppercase field names (as expected) but 'field.getName()' gives a lowercase name.

Please see this forum thread:
http://www.eclipse.org/forums/index.php/mv/msg/487418/1058520/#msg_1058520
